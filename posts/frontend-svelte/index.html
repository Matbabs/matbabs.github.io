Il s’agit d’un framework orienté composant qui devient de plus en plus populaire vis-à-vis de React, Angular et VueJs.

Un compilateur plus qu’un framework#

Un framework front orienté composant.

Cependant, il faut voir celui-ci comme un compilateur et pas seulement un framework :

  • Optimise vos instructions JS
  • Taille bundle largement réduite
  • Meilleure réactivité au runtime (pas de DOM)

Top en GREEN IT car plus léger

Historique : son créateur Rich Harris#

Svelte est crée par Rich Harris. Son créateur avait fait le postulat suivant:

“Les frameworks front modernes ont trop de travail au runtime”

En 2019 Svelte v3 est né, il apporte de réelles nouveautés sur la réactivité et la simplification du code.

Organisation des composants#

ex: MyComponent.svelte

<!--script principal-->
<script>
    //imports
    import MyOtherComponent from "./MyOtherComponent.svelte"
    //props
    export let myProps = "default value"
    //variables
    let name = "John"
    let inputVariable = ""
</script>

<!--style-->
<style>
    p {
        color: red
    }
</style>

<!--template principal HTML-->
<h1>My App</h1>
<p> {name} </p>
<MyOtherComponent myOtherProps={42}/>
<input type="text" bind:value={inputVariable}> <!--binding with inputVariable-->

Nous observons une écriture ultra simplifiée en comparaison des équivalents React & Angular.

La réactivité#

La réactivité standard par ACTION UTILISATEUR :

  • -> action utilisateur
    • -> mise à jour d’état
      • -> détection des changements (algo de calcul qui est couteux)
        • -> application de ces changements.

Dans React & VueJS nous utilisons le virtual DOM (qui est normalement suffisamment rapide). Cependant, si nous avons un besoin fort en performance (page très dynamique, animations, etc.), il peut montrer des faiblesses.

La réactivité cotée Svelte par ASSIGNEMENT DE VARIABLE :

  • -> une variable est (ré)-assigné (il est possible de déterminer les changements nécessaires)
    • -> application des changements

Svelte se charge de faire des optimisations lors de la phase de build. Il se base sur l'assignement de variable.

Nous utilisons le symbole $: à la place du let. Permet d’imposer le changement d’état sur la variable assignée (équivalence syntaxique des hooks useState/useEffect de React mais la réactivité est fondamentalement différente).

Les stores#

Permet de gérer notre état de données global dans notre application. Un équivalent d’une gestion d’un store Redux en React sans actions et reducers cependant.

  • Writable: Écriture de valeur dans nos variables
  • Readable: Lecture des valeurs de nos variables
  • Derived: A l’écoute des autres stores

Svelte-KIT (outillage autour de Svelte)#

Ajoute des éléments de performance et d’outillage :

  • HMR avec Vite
  • Routing (mise en place de endpoints pour le SSR)
  • Server Side Rendering (SSR)
  • CLI
  • Adapters (outils pour le déploiement)

svelte-kit package permet d’exporter tous nos composants issus de ./lib dans une librairie (pratique pour créer une lib de composants graphique par exemple).
